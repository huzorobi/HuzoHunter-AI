# Contributing to HuzoHunter AI

Thanks for your interest in HuzoHunter AI! The project is in an early **research and
development** phase, and contributions, ideas, and feedback are all welcome.

## Ways to Contribute

- 💡 **Ideas & feedback** — open an [Issue](../../issues) to suggest features or improvements.
- 🐛 **Bug reports** — describe what you expected, what happened, and how to reproduce it.
- 📖 **Documentation** — improvements to docs are always appreciated.
- 🔧 **Code** — see below.

## Getting Started

1. Fork the repository and clone your fork.
2. Create a feature branch: `git checkout -b feature/your-idea`.
3. Make your changes with clear, focused commits.
4. Push and open a Pull Request describing the change and its motivation.

## Guidelines

- Keep pull requests small and focused on a single concern.
- Write clear commit messages (imperative mood, e.g. "Add OpenSearch ingest mapping").
- Update relevant documentation when behaviour changes.
- Be respectful and constructive in all discussions.

## Security Issues

**Do not** open public Issues for security vulnerabilities. Please follow the process in
[SECURITY.md](SECURITY.md) instead.
